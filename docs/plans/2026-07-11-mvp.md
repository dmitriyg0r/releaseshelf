# ReleaseShelf MVP Implementation Plan

> **For Hermes:** Implement this plan in small, test-first vertical slices.

**Goal:** Build a Tauri desktop catalog for GitHub release assets that recognizes macOS and Linux packages and gives the user a transparent, safe download action.

**Architecture:** React/TypeScript renders the catalog and calls narrowly scoped Rust Tauri commands. Rust owns package classification and later will own GitHub access, download resumption, integrity checks, and OS hand-off. The first UI uses curated local data so the user experience can be validated before GitHub API and OAuth are introduced.

**Tech Stack:** Tauri 2, Rust, React 19, TypeScript, Vite, CSS; native Rust unit tests and Vitest for UI behavior.

---

### Task 1: Bootstrap and validate the desktop project

**Files:** generated Tauri scaffold; `src-tauri/tauri.conf.json`

1. Create a Tauri 2 React TypeScript project named `releaseshelf`.
2. Initialize Git and record the baseline.
3. Run `npm run build` and `cargo test --manifest-path src-tauri/Cargo.toml`.

### Task 2: Classify release assets in Rust

**Files:**
- Modify: `src-tauri/src/lib.rs`
- Test: unit tests in `src-tauri/src/lib.rs`

1. Write a test that expects macOS DMG, macOS PKG, Linux AppImage, DEB, RPM, archive, and unknown assets to be distinguished.
2. Run `cargo test` and confirm the test fails because the classifier does not exist.
3. Implement the smallest enum/function that satisfies the test.
4. Expose it through a typed Tauri command only after the unit test passes.

### Task 3: Deliver a catalog UI vertical slice

**Files:**
- Modify: `src/App.tsx`, `src/App.css`, `src/index.css`
- Create: `src/catalog.ts`, `src/catalog.test.ts`

1. Write a failing TypeScript test for platform-aware asset selection from curated release data.
2. Implement the pure selector and run the test.
3. Build a browseable catalog with a search field, platform filter, application cards, and a release details pane.
4. Use only local curated data; no unverified install or download action.

### Task 4: Integrate native asset classification

**Files:** `src/App.tsx`, `src-tauri/src/lib.rs`

1. When a user selects an asset, ask Rust to classify it and render the result.
2. Make the primary action say “Open release on GitHub” rather than install files.
3. Validate the frontend build and native test suite.

### Task 5: Next implementation milestone

Add a GitHub releases client with ETag/rate-limit handling, then a resumable download manager with checksum verification. Do not automate installation or bypass Gatekeeper/Linux package policy.
